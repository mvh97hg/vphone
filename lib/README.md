# VPhone Library

`vphone` provides two integration layers:

- A DOM widget that embeds the existing VPhone UI into web pages or Electron renderers.
- A typed headless SDK facade for projects that want to build their own UI.

## Files

| File | Purpose |
| --- | --- |
| `vphone.js` | UMD widget runtime for browser, CommonJS, and ESM consumers. |
| `vphone.min.js` | Production single-file build generated by `npm run build:release`. |
| `dist/vphone-sdk.*` | ESM, CommonJS, and TypeScript declaration output for the headless SDK. |
| `react-native.js` | React Native/CommonJS entrypoint for the headless core SDK (`createPhoneClient`). |
| `src/` | SDK TypeScript source. |
| `LIBRARY_API.md` | Full API reference. |
| `CLICK_TO_CALL.md` | Click-to-call integration guide. |
| `EMBEDDED_UI_GUIDE.md` | Embedded UI notes. |

## Install Locally

```bash
cd lib
npm install
npm test
npm run build:release
```

## Widget Usage

### Floating Widget

```html
<script src="lib/vphone.js"></script>
<script>
  const phone = Vphone.mount({
    phoneUrl: 'Phone/index.html',
    floating: true,
    width: '250px',
    height: '425px',
    config: {
      wssServer: 'pbx.example.com',
      WebSocketPort: 7443,
      ServerPath: '/ws',
      SipDomain: 'pbx.example.com',
      SipUsername: '1001',
      SipPassword: 'secret',
      Language: 'vi',
      EnableVideoCalling: false,
      MaxDidLength: 24
    }
  });
</script>
```

### Inline Panel

```html
<div id="phone-panel" style="width:250px;height:425px"></div>
<script src="lib/vphone.js"></script>
<script>
  Vphone.mount({
    phoneUrl: 'Phone/index.html',
    container: '#phone-panel',
    floating: false,
    width: '250px',
    height: '425px',
    config: {
      wssServer: 'pbx.example.com',
      WebSocketPort: 7443,
      ServerPath: '/ws',
      SipDomain: 'pbx.example.com',
      SipUsername: '1001',
      SipPassword: 'secret',
      DisableBuddies: true,
      DisableFreeDial: false
    }
  });
</script>
```

## Headless SDK Usage

```js
import { createPhoneClient } from 'vphone/sdk';
// or: import { createPhoneClient } from 'vphone/core';
// or RN/CJS: const { createPhoneClient } = require('vphone/react-native');

const phone = createPhoneClient({
  sip: {
    wssServer: 'pbx.example.com',
    webSocketPort: 7443,
    serverPath: '/ws',
    sipDomain: 'pbx.example.com',
    username: '1001',
    password: 'secret'
  },
  media: { audio: true, video: false },
  language: 'vi'
});

phone.on('registration', event => console.log(event.state));
await phone.register();
await phone.makeCall('1002', { callType: 'audio' });
```

`createPhoneClient()` is DOM-independent and can be used in React Native (or other mobile runtimes) as a core signaling/call-state layer with a custom platform adapter.

The SDK accepts adapter hooks, so a web app, Electron app, or future React Native bridge can supply its own SIP/WebRTC implementation while keeping one call-control API.

For DOM UI bridge use:

```js
import { createPhoneClient, createIframeBridgeAdapter } from 'vphone/sdk';
import Vphone from 'vphone/widget';

const widget = Vphone.mount({ container: '#phone-panel', floating: false, config: {/* ... */} });
const core = createPhoneClient({
  sip: { wssServer: 'pbx.example.com', webSocketPort: 7443, username: '1001', password: 'secret' }
}, { adapter: createIframeBridgeAdapter(widget) });
```

## Mount Config

| Option | Default | Description |
| --- | --- | --- |
| `phoneUrl` | `embedded` | External URL for `Phone/index.html`; use this for web/Electron hosting. |
| `container` | `null` | CSS selector or element for inline mode. |
| `floating` | `!container` | `true` for launcher widget, `false` for inline. |
| `width` | `250px` | Panel width. |
| `height` | `425px` | Panel height. |
| `title` | `VPhone` | Widget title. |
| `showStatus` | `true` | Show status indicator in widget chrome. |
| `startCollapsed` | `true` | Floating widget starts collapsed. |
| `expandOnIncomingCall` | `true` | Expand widget when an incoming call event arrives. |
| `config` | `{}` | Phone provisioning/config object. |
| `onStatusChange` | `null` | Registration/status callback. |
| `onCallEvent` | `null` | Call lifecycle callback. |

## Common Phone Config Toggles

Pass these under `Vphone.mount({ config: { ... } })`.

| Config | Default | Description |
| --- | --- | --- |
| `EnableVideoCalling` | `false` | Show/use video call features. |
| `EnableTextMessaging` | `false` | Enable SIP text messaging UI. |
| `EnableTransfer` | `true` | Enable attended/blind transfer controls. |
| `EnableConference` | `true` | Enable conference controls. |
| `EnableAlphanumericDial` | `false` | Allow letters in dial input. |
| `EnableRingtone` | `true` | Enable inbound ringtone. |
| `DisableFreeDial` | `false` | Hide free dial/dialpad entry. |
| `DisableBuddies` | `false` | Hide manual buddy management. |
| `RecordAllCalls` | `false` | Start call recording automatically. |
| `AutoAnswerPolicy` | `allow` | `allow`, `disabled`, or `enabled`. |
| `DoNotDisturbPolicy` | `allow` | `allow`, `disabled`, or `enabled`. |
| `CallWaitingPolicy` | `allow` | `allow`, `disabled`, or `enabled`. |
| `CallRecordingPolicy` | `allow` | `allow`, `disabled`, or `enabled`. |
| `HideSettingsButton` | `true` in embed | Hide settings button. |
| `HideRecordAllCallsButton` | `true` in embed | Hide record-all-calls button. |
| `MaxDidLength` | `24` | Maximum dial input length. |
| `KeyboardShortcuts` | F2/Escape/F4/F6/F8/F9 | Remap answer, hangup, hold, mute, transfer, and dialpad shortcuts. |

Example:

```js
Vphone.mount({
  phoneUrl: 'Phone/index.html',
  floating: true,
  config: {
    EnableVideoCalling: true,
    EnableTransfer: false,
    EnableConference: false,
    EnableTextMessaging: false,
    CallRecordingPolicy: 'disabled',
    RecordAllCalls: false,
    AutoAnswerPolicy: 'disabled',
    DisableBuddies: true,
    MaxDidLength: 24,
    KeyboardShortcuts: {
      answer: 'F2',
      hangup: 'Escape',
      hold: 'F4',
      mute: 'F6',
      transfer: 'F8',
      dialpad: 'F9'
    }
  }
});
```

## SIP And Media Config

| Config | Description |
| --- | --- |
| `wssServer` | SIP WebSocket host. |
| `WebSocketPort` | SIP WebSocket port. |
| `ServerPath` | WebSocket path, for example `/ws`. |
| `SipDomain` | SIP domain/realm. Defaults to `wssServer` when empty. |
| `SipUsername` | Extension/SIP username. |
| `SipPassword` | SIP password or HA1 hash. |
| `SipPasswordType` | `plain` or `ha1`. |
| `IceStunServerJson` | JSON string for STUN/TURN servers. |
| `IceStunCheckTimeout` | ICE gathering timeout in milliseconds. |
| `MaxVideoBandwidth` | Outgoing video bandwidth in Kb/s; `-1` disables limit. |
| `videoHeight` | Preferred camera height. |
| `maxFrameRate` | Preferred camera frame rate. |
| `videoAspectRatio` | Preferred camera aspect ratio. |
| `MediaConfig` | Optional media override map for `ringtone`, `holdMusic`, `busy`, `callWaiting`, `congestion`, and `earlyMedia`. `holdMusic` is optional and only plays when explicitly configured. Legacy alias `ringbackTone` is still accepted for `earlyMedia`. |

Media override example:

```js
config: {
  MediaConfig: {
    ringtone: 'https://cdn.example.com/media/ringtone.mp3',
    holdMusic: 'https://cdn.example.com/media/hold.mp3',
    busy: 'Busy.mp3',
    callWaiting: 'CallWaiting.mp3',
    congestion: 'Congestion.mp3',
    earlyMedia: 'EarlyMedia.mp3'
  }
}
```

When users change media in Settings, values are persisted in local storage and override app defaults until cleared.

```js
config: {
  IceStunServerJson: JSON.stringify([
    { urls: 'stun:stun.l.google.com:19302' },
    { urls: 'turn:turn.example.com:3478', username: 'user', credential: 'pass' }
  ])
}
```

## Build Output

`npm run build:release` runs:

- `build:sdk`: emits `dist/vphone-sdk.mjs`, `.cjs`, and `.d.ts`.
- `build:single`: embeds the phone UI into `vphone.min.js`.

Run `npm test` before publishing.

## Requirements

- Secure origin: HTTPS, localhost, or secure Electron renderer.
- SIP WebSocket server with WebRTC/DTLS-SRTP support.
- Microphone/camera permissions for audio/video calls.
